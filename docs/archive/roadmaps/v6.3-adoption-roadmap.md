# V6.3 Adoption Roadmap (Developer + Agent UX)

Status: Completed (March 23, 2026)  
Date: March 21, 2026  
Scope: Post-V6.2 stabilization and usability hardening

## Why This Roadmap

A local end-to-end run was validated on `next-blogkit-pro` using the local CLI binary:

```powershell
node E:\Web Project\experimental-workspace\apex-auditor-workspace\signaler\dist\bin.js <command>
```

Observed baseline (March 21, 2026):

- `discover` (quick): ~8.6s
- `run` (v3 throughput, 24 combos): ~300.6s
- `analyze` (v6 lean): ~2.4s
- `verify` (v6 top-actions=2 throughput): ~237.6s (exit `2`, checks failed)
- `report`: ~3.9s

The core flow is working, but adoption quality is now limited by UX consistency and loop efficiency details.

## V6.3 Goals

1. Remove command UX ambiguities that slow first-time and scripted usage.
2. Make agent contracts explicit even when optional inputs are not enabled.
3. Reduce full-loop cost for large projects without reducing trust.

## Workstream F: CLI UX Hardening

Status: Planned

- [x] Make `--help` behavior side-effect-free for every command (`discover`, `run`, `analyze`, `verify`, etc.).
- [x] Ensure command-specific help exits `0` and never prompts or starts network/audit work.
- [x] Normalize suggested commands so output matches invocation context (installed `signaler` vs local `node .../dist/bin.js`).
- [x] Remove mojibake/encoding artifacts from Windows CLI output and error surfaces.

Exit criteria:

- [x] `node dist/bin.js discover --help` prints help only, no prompt, exit `0`.
- [x] `node dist/bin.js run --help` prints run options only, no audit side effects, exit `0`.
- [x] No malformed Unicode glyphs in standard error paths on Windows.

## Workstream G: Agent Contract Clarity

Status: Planned

- [x] Always emit `externalSignals` metadata in `suggestions.json` and `analyze.json`.
- [x] Preserve additive backward compatibility (no required-field removals).
- [x] Keep ranking policy metadata explicit (`version`, `formula`) and consistent across outputs.
- [x] Add deterministic metadata digest/counters for enabled and disabled external-input paths.

Exit criteria:

- [x] No-flag runs include `externalSignals.enabled=false` with zero counters.
- [x] Flag-enabled runs include accepted/rejected counters and digest.
- [x] Existing v3/v6 consumers continue to parse without migration changes.

## Workstream H: Loop Efficiency and Cost Controls

Status: Implemented

- [x] Add runtime-aware verify planning so route selection can be constrained by target budget.
- [x] Improve low-memory throughput messaging with explicit "speed vs stability" guidance in command output.
- [x] Emit compact per-command timing metadata for `analyze`/`verify` summaries to aid agent orchestration.
- [x] Document local-source execution path as a first-class workflow for unpublished builds.

Exit criteria:

- [x] Verify route planning can be bounded by explicit runtime budget.
- [x] Agent loops can estimate expected runtime before executing focused reruns.
- [x] Docs include copy/paste local execution commands for workspace testing.

## Workstream I: Success Gate Automation

Status: Implemented

- [x] Add deterministic V6.3 success-gate evaluator (`bench:v63:gate`) with blocking vs warn-only checks.
- [x] Add tiny canonical loop smoke evidence runner (`bench:v63:loop`) for reproducible end-to-end timing/evidence.
- [x] Add report validator (`bench:v63:validate`) and schema for CI/local verification.
- [x] Add focused validator unit tests and docs for running the V6.3 gate.
- [x] Track remaining evidence as warn-only/manual instead of implicit tribal knowledge.

Exit criteria:

- [x] V6.3 adoption readiness can be measured via machine-readable report artifacts.
- [x] Blocking failures are explicit and reproducible in local + CI flows.
- [x] Success-gate docs are linked from canonical CLI/CI and docs index pages.
